## 问题1：请谈一下对keep-alive的理解？

`keep-alive`是vue中的内置组件，能在组件切换过程中将状态保留在内存中，**防止重复渲染DOM**。

`keep-alive` 包裹动态组件时，会缓存不活动的组件实例，而不是销毁它们

:::info keep-alive 对生命周期的核心改造
设置了 `keep-alive` 缓存的组件，会多出两个生命周期钩子（activated与deactivated）：
- 首次进入组件时：beforeRouteEnter > beforeCreate > created> `mounted` > `activated` > ... ... > beforeRouteLeave > `deactivated`
- 再次进入组件时：beforeRouteEnter >`activated` > ... ... > beforeRouteLeave > `deactivated`
:::

`keep-alive`可以设置以下props属性：
- `include `- 字符串或正则表达式。只有名称匹配的组件会被缓存
- `exclude` - 字符串或正则表达式。任何名称匹配的组件都不会被缓存

**使用场景:**
- 缓存多标签应用(Tabs)
- 路由组件缓存:路由页面频繁切换


**底层逻辑**
::: info 底层逻辑
- **缓存存储**:`keep-alive` 内部维护一个 `cache` 对象（key = 组件标识，value = 组件实例）和 keys 数组（记录缓存顺序）

- **缓存判断**：首次渲染组件时，若**匹配include**/**不匹配exclude**，就将组件实例存入 `cache`
:::

**常见误区**
:::tip
- **误区1**：keep-alive 缓存了 DOM，所以数据不变

 **真相：缓存的是组件实例（包含数据、状态），DOM 只是被隐藏（display: none），数据不变是因为实例没被重建**

- **误区2**：路由参数不同的同一页面，缓存会自动区分

真相：默认按组件 name 缓存，若两个路由对应同一组件（如 /detail/1 和 /detail/2），会共享缓存导致数据冲突；
解决：给 `<router-view>` 设置唯一 key：
```js
<keep-alive>
  <router-view :key="$route.fullPath"></router-view>
</keep-alive>
```
:::


## Vue响应式原理
>参考内容

**宏观层面**:Vue3 的响应式核心思想是 **依赖收集** 和 **派发更新** 。它利用 ES6 的 Proxy 对数据进行拦截，当数据被读取时收集依赖（Track），当数据被修改时触发依赖（Trigger）

**微观实现**:具体来说，它维护了一个全局的 `WeakMap （targetMap）`，用来建立 对象 -> 属性 -> 副作用函数(Set) 的映射关系。
- 在 get 拦截中，如果当前有激活的副作用（activeEffect），就把它存入对应的 Set 中。
- 在 set 拦截中，从 Map 中取出对应的副作用集合，遍历执行。”
