## 问题1：Vue2和vue3的主要区别？

| 维度            | Vue2                                 | Vue3                                                                                            |
| --------------- | ------------------------------------ | ----------------------------------------------------------------------------------------------- |
| 响应式原理      | Object.defineProperty（劫持属性）    | Proxy（代理对象）                                                                               |
| 组合式 API      | 仅 Options API（data/methods/watch） | 新增 Composition API（setup/reactive/ref），兼容 Options API                                    |
| 虚拟 DOM / 编译 | 未做针对性优化，编译逻辑简单         | 重构虚拟 DOM，编译阶段做 PatchFlags / 静态提升，性能提升～50%                                   |
| 生命周期        | beforeCreate/create 等选项式         | setup 替代 beforeCreate/create，新增 onRenderTracked 等，生命周期钩子改为函数式（如 onMounted） |
| 性能优化        | 无 Tree-Shaking，体积较大            | 模块化设计，Tree-Shaking 友好，按需引入，体积更小                                               |

## 问题2：vue2和vue3的响应式原理？

**Vue2 痛点**：
`Object.defineProperty` 只能劫持对象的**已有属性**，无法监听数组下标修改、对象新增 / 删除属性；需通过 `$set/$delete` 手动触发响应式，且对数组的监听需重写 push/pop 等方法，逻辑复杂。

**Vue3 优化：**
Proxy 代理整个对象，能**监听对象新增 / 删除属性、数组下标修改、Map/Set 等集合类型，无需手动触发**，响应式更完整。

:::code-group
```js [Vue2]
// Vue2 响应式示例（需手动处理）
export default {
  data() {
    return {
      obj: { name: '张三' },
      arr: [1,2,3]
    };
  },
  mounted() {
    this.obj.age = 18; // 新增属性，无响应式
    this.$set(this.obj, 'age', 18); // 需手动 $set
    this.arr[0] = 10; // 数组下标修改，无响应式
    this.arr.push(4); // 重写的方法，有响应式
  }
};
```
```js [vue3优化]
// Vue3 响应式示例（自然响应式）
import { reactive } from 'vue';
export default {
  setup() {
    const obj = reactive({ name: '张三' });
    const arr = reactive([1,2,3]);
    
    obj.age = 18; // 新增属性，自动响应式
    delete obj.name; // 删除属性，自动响应式
    arr[0] = 10; // 数组下标修改，自动响应式
    return { obj, arr };
  }
};
```
:::

## 问题3:vue的生命周期？

![](/vue/生命周期.png)

::: info 生命周期详情
生命周期并非指 Vue 中的生命周期钩子，而是指 Vue 内部从初始化到挂载 DOM 的完整流程。从开始创建、初始化数据、编译模版、挂载 Dom -> 渲染、更新 -> 渲染、卸载等一系列过程，我们称这是 Vue 的生命周期。
:::

## 问题4:data 为什么是一个函数而不是对象?
在 Vue 组件中，data 必须是**函数而非对象**，核心原因是**避免多个组件实例共享同一份数据**：如果 data 是对象，所有实例会引用同一个内存地址的对象，修改一个实例的属性会影响其他实例；而 data 作为函数时，每次创建组件实例都会执行函数并返回新的独立对象，保证各实例数据隔离。



## 5.vue双向绑定原理

Vue 的双向绑定核心是 “数据劫持 + 发布 - 订阅模式”：Vue2 基于 `Object.defineProperty` 劫持对象属性的 get/set 方法，Vue3 改用 Proxy 代理整个对象，二者均在数据变更时触发 `set/Proxy.set`，通知依赖的视图更新；

同时结合 `v-model` 指令，在表单输入事件中反向修改数据，最终实现 “数据变→视图更，视图变→数据更” 的双向联动。

## 6.v-if和v-for优先级?

`Vue2` 中 v-for 的优先级高于 v-if，这意味着渲染时会先遍历数组 / 对象，再对每个项执行 v-if 判断，容易导致性能浪费；
`Vue3` 则调整为 v-if 优先级高于 v-for，若二者写在同一元素上会直接报错（强制开发者避免这种写法）
核心原则：永远不要将 v-if 和 v-for 写在同一个元素上，应将 v-if 提至父元素控制整体渲染，或用计算属性先过滤数据再遍历。
## 7.v-if和v-for的区别？

- **控制手段**

`v-show`隐藏则是为该元素添加`css--display:none`，dom元素依旧还在
`v-if`显示隐藏是将dom元素整个添加或删除

- **编译条件**
  - **v-if是真正的条件渲染**，它会确保在切换过程中条件块内的事件监听器和子组件适当地被销毁和重建。只有渲染条件为假时，并不做操作，直到为真才渲染
  - v-show 由false变为true的时候不会触发组件的生命周期
  - v-if由false变为true的时候，触发组件的beforeCreate、create、beforeMount、mounted钩子，由true变为false的时候触发组件的beforeDestory、destoryed方法
