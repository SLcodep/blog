**React 组件本质**：UI + update + 常规的类和函数 = React 组件。组件承载了渲染视图的 UI 和更新视图的 setState 、 useState 等方法。React 在底层逻辑上会像正常实例化类和正常执行函数那样处理的组件。Component 加入 updater, 通过 updater 控制 state 和更新视图。组件的数据结构本质是一个函数或者类，当你使用元素标签的方式进行调用时，函数或者类会被执行，最终返回一个 React 元素。

## React 组件
**React 组件是一段可以 使用标签进行扩展 的 JavaScript 函数**。react允许你将标签、CSS 和 JavaScript 组合成自定义组件，即 **应用程序中可复用的 UI 元素**。

React有两种核心组件形式:**类组件（Class Component）**（React 16.8 前的主流）和 **函数组件（Function Component）**（现在的推荐写法）。

> 类组件是 ES6 类的形式，继承 React.Component，是 Hooks 出现前唯一能拥有状态的组件，现在新项目已很少用，但老项目仍会见到。

## 组件通信
组件通信大致分为几种方式：
1.父子组件(父传子，子传父)
2.兄弟组件通信
3.跨层通信context
4.状态管理库redux/Zustand

### 1.父子组件

#### 父传子

这是最常用的通信方式，父组件通过**属性**给子组件传值 / 传方法，子组件通过 `props` 接收。

```js
import React, { useState } from 'react';

// 子组件：接收父组件的 props
function Child(props) {
  return (
    <div>
      <p>子组件接收的父组件数据：{props.message}</p>
      {/* 子组件调用父组件传过来的方法 */}
      <button onClick={() => props.onChildClick('子组件的消息')}>
        点击给父组件发消息
      </button>
    </div>
  );
}

// 父组件：传递 props 给子组件
function Parent() {
  const [parentMsg, setParentMsg] = useState('初始消息');
  // 定义给子组件调用的方法
  const handleChildClick = (msg) => {
    setParentMsg(`父组件收到：${msg}`);
  };

  return (
    <div>
      <p>父组件状态：{parentMsg}</p>
      {/* 传数据 + 传方法 */}
      <Child message="我是父组件的消息" onChildClick={handleChildClick} />
    </div>
  );
}
```
下面介绍一些props的常见操作:
**1.解构赋值简化 props 使用**
通常你不需要整个 props 对象，所以可以将它解构为单独的 props
:::tip
在声明props时， 不要忘记(和)之间的一对花括号{和}
```js
function Avatar({ person, size }) {
  // ...
}
```
这种语法被称为 “解构”，等价于于从函数参数中读取属性：
```js
function Avatar(props) {
  let person = props.person;
  let size = props.size;
  // ...
}
```
:::

**2.透传 props（剩余属性）**
当父组件需要把多个 props 传递给子组件，且不想逐个列举时，用剩余运算符 `...` 透传，适合封装通用组件。
```js
// 父组件：传递多个 props 给子组件
function Parent() {
  const [data, setData] = useState('初始数据');
  const commonProps = {
    data,
    className: 'brother-a-style',
    title: '兄弟A数据展示',
  };

  return (
    <div>
      <BrotherA {...commonProps} /> {/* 透传所有 commonProps */}
      <BrotherB onChangeData={setData} />
    </div>
  );
}
```

#### 子传父
React遵循单向数据流:props 是只读的，子组件不能直接修改 props，必须通过父组件传递的回调函数修改父组件的状态。

如上面父传子代码中的示例，通过` <button onClick={() => props.onChildClick('子组件的消息')}>`回调函数修改父组件的状态。

### 2.兄弟组件通信
兄弟组件没有直接关联，需要借助**共同的父组件**作为中间层：

1.父组件定义状态和修改状态的方法
2.父组件把状态传给子组件 A，把修改方法传给子组件 B
3.子组件 B 调用方法修改父组件状态，父组件再把新状态传给子组件 A，实现兄弟通信

```js
import React, { useState } from 'react';

// 兄弟1：展示数据
function BrotherA(props) {
  return <div>兄弟A展示：{props.data}</div>;
}

// 兄弟2：修改数据
function BrotherB(props) {
  return (
    <button onClick={() => props.onChangeData('兄弟B修改的数据')}>
      点击修改兄弟A的数据
    </button>
  );
}

// 父组件：作为中间层
function Parent() {
  const [data, setData] = useState('初始数据');

  return (
    <div>
      <BrotherA data={data} />
      <BrotherB onChangeData={setData} />
    </div>
  );
}

export default Parent;
```
